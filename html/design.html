map:

dataMap, a map, is defined in lines 10-14 inclusive in the file CW3/js/visit.js. Since the name of each key is known and immediately points us to its value, it is more efficient than using, for example, an array to store visitorNum, date, and time. The map was especially useful when arguments were to be passed to the reserve() function - I did not have to write extra code to determine which data value had to go as which parameter.

Overall, there is a clear distinction between structure, style, and script.

How styles are grouped:

Both the embedded and style sheet method has been used. All the folders - except for (CW3/html/home_page.html), which uses embedded style - have style sheets linked to them.

A separate style sheet (CW3/css/all.css) has been used to apply the header and footer styles and animation to every page.

How scripts are grouped:

Both the inline and linked files method have been used. Embedded style was not used since it did not seem useful or sensible for this project.
